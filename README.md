# 💰 思源笔记理财监控插件

一个专业的理财数据监控插件，支持多接口配置、实时价格预警、历史数据可视化等功能。

![preview](preview.png)

## ✨ 功能特性

### 📊 历史数据展示
- 点击顶栏 📈 按钮打开数据面板
- 实时统计：当前价格、最高价、最低价、涨跌幅
- 可视化价格走势图（支持 1天/7天/30天/全部 时间范围）
- 详细历史数据表格，最多保留最近 1000 条记录

### 🔔 智能价格预警
支持四种预警类型，触发时通过思源通知提醒：

| 预警类型 | 说明 | 示例 |
|---------|------|------|
| **价格上涨至** | 价格达到设定值时提醒（避免重复提醒） | 黄金涨至 950 元/克时提醒 |
| **价格下跌至** | 价格低于设定值时提醒（避免重复提醒） | 黄金跌至 900 元/克时提醒 |
| **日跌幅超%** | 从当天最高价下跌超过设定百分比时提醒 | 从最高价下跌 3% 时提醒 |
| **价格变化超%** | 相比上次查询变化超过设定百分比时提醒 | 单次波动超过 2% 时提醒 |

### 🔌 多接口支持
- 支持配置多个 API 接口
- 每个接口可独立启用/禁用
- 每个接口可监控多个产品类型
- 接口间数据相互独立

### ⏰ 自动查询
- 每 30 分钟自动查询一次（整点和半点）
- 支持手动刷新数据
- 自动保存历史数据


## 🚀 使用指南

### 第一步：配置 API

默认已添加极速数据黄金 API，但需要填写您的 **appkey** 才能使用。

1. 打开插件设置面板
2. 切换到"接口配置"标签页
3. 点击"极速数据-黄金"的"编辑"按钮
4. 在"API密钥 (appkey)"处填写您的极速数据 appkey
5. 点击"保存"

> 💡 **获取 appkey**：访问 [极速数据](https://www.jisuapi.com/) 注册账号并申请黄金数据 API 的 appkey（免费版通常有每日调用次数限制）

### 第二步：设置预警规则

1. 在接口编辑界面，点击产品列表旁的"预警规则"按钮
2. 根据需求设置预警条件（留空则不启用该预警）
3. 点击"保存规则"

### 第三步：查看数据

1. 点击思源笔记顶栏的 📈 按钮
2. 选择要查看的接口和产品
3. 查看实时价格、走势图和历史数据

## ⚙️ 详细配置说明

### 常规设置

| 设置项 | 说明 | 默认值 |
|-------|------|--------|
| 启用自动查询 | 是否开启定时自动查询 | 开启 |
| 查询间隔 | 自动查询的时间间隔（分钟） | 30 分钟 |

### 接口配置

| 设置项 | 说明 | 示例 |
|-------|------|------|
| 接口名称 | 自定义接口的显示名称 | 极速数据-黄金 |
| API地址 | 数据接口的 URL | `https://api.jisuapi.com/gold/bank` |
| API密钥 | 接口认证密钥（如 appkey） | 您的极速数据 appkey |
| 启用此接口 | 是否启用该接口的查询 | 开启 |

### 预警规则

每个产品可以独立设置预警规则：

- **价格上涨至**：当价格 **≥** 设定值时触发一次提醒
- **价格下跌至**：当价格 **≤** 设定值时触发一次提醒
- **日跌幅超百分之几**：计算公式为 `(最高价 - 当前价) / 最高价 * 100%`
- **价格比上次涨幅超百分之几**：计算公式为 `(当前价 - 上次价格) / 上次价格 * 100%`

## 📡 接入其他数据源

插件设计为通用接口模式，可接入任何返回 JSON 格式的价格数据 API。

### 数据格式要求

API 需返回以下格式的数据：

```json
{
  "status": 0,
  "msg": "ok",
  "result": [
    {
      "typename": "产品名称",
      "midprice": "中间价",
      "buyprice": "买入价",
      "sellprice": "卖出价",
      "maxprice": "最高价",
      "minprice": "最低价",
      "updatetime": "更新时间"
    }
  ]
}
```

### 配置步骤

1. 在设置面板中点击"+ 添加接口"
2. 填写接口名称、API 地址和密钥（如有）
3. 点击"+ 添加产品"，输入要监控的产品名称（需与 API 返回的 `typename` 一致）
4. 设置该产品的预警规则
5. 保存配置

## 🔒 隐私说明

- 所有历史数据保存在本地，不会上传到任何服务器
- API 密钥（appkey）保存在本地设置中
- 仅向您配置的 API 地址发送请求

## ❓ 常见问题

### Q: 为什么收不到预警通知？
A: 请检查：
1. 预警规则是否已设置且留空不启用
2. 接口是否已启用
3. appkey 是否正确填写
4. 思源笔记的通知权限是否开启

### Q: 如何清空历史数据？
A: 在设置面板的"接口配置"中，点击对应接口卡片底部的"清空历史数据"链接。

### Q: 可以同时监控多个产品吗？
A: 可以。每个接口可以添加多个产品，每个产品有独立的预警规则。

### Q: 预警会重复提醒吗？
A: 不会。价格上涨/下跌预警会在条件首次满足时提醒，之后价格回到阈值以下/以上后再次满足条件才会再次提醒。
